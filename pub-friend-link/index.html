    友情链接交换的开放提议

    友情链接，简称友链。从字面意义上理解，在个人网站的范畴里，这应当指与自己具有一定友情的他人的网站链接。

    我比较认同这种建立在已有的关系基础之上的友链。自然，本页也不适用于这种友链，更不是交换友链的唯一途径；至于其他途径则看缘分，并没有具体安排。

    但也存在一种公开邀约而交换的友情链接。这种状况就像是现实中的人们先交换联系方式、打算在之后再培养友谊一样。

    对此，我也持开放态度，所以才有了本页；但如果只是单纯地挂个链接和简介，我想这并没有体现出“友情”，而只是“链接”。1

    所以，如果你想通过本页所述方式来交换友链的话，请允许我提出一个特殊的要求：

    请向本空间站投稿一篇原创文章，通过之后即可交换友链。

    以文交友，不亦乐乎？

    需要说明的是，虽然本页看起来很复杂，但实际的要求其实是很普通的，一言以蔽之就是“正常”即可。2

    所要求提供的资料也绝大多数都是现成的；此外，这里的投稿也并非是单向的……

    总之，看下面的细节你就能明白了。

    细则

    请你根据以下各节说明，发送所需资料至 clsty@celestialy.top。

    总体要求

    对个人网站的要求：
    • 最好是有独立域名的网站，但是域名不能一眼看上去就像违法网站（比如 <一串随机数字>.<某个顶级域名> ）。
    • 如果是 GitHub、博客园这种相对干净的平台所提供的个人博客或“子网站”，也可以接受。
    • 不接受 CSDN、知乎、Bilibili 等平台的个人“博客”或“子网站”。
    • 不接受在国内几乎完全无法正常访问的网站；类似 GitHub 这种时而抽风但仍可直接访问的除外。
    对稿件的要求：
    • 投稿的内容质量，“尚可”即可，最好是技术向但并不限定于此。
    • 你必须是所投稿文章的唯一的原创作者，且持有版权3，并且你向我投稿并不违反你所涉及的合同、约定、法规等。在此前提下，你可以一稿多投，比如你可以直接用个人网站里现成的文章来投稿。
    其他说明：
    • 请不要发送含有敏感/暴恐/色情/侵权/其他违法内容的任何资料（包括投稿、个人资料、个人网站资料等）。
    • 如果我感到我与你发送的资料（包括个人网站的内容）格格不入（比如较大量地涉及饭圈、广告、民科等），也可能会拒绝友链交换。当然这并不意味着对你的否定，我们只是在做双向选择而已。

    资料之一：投稿文章

    文章
    Markdown 格式的文件（不需要导言区）。
    • 不可以在 Markdown 文件里嵌入需要从外部网站加载的资源，比如图片、CSS、JS 等。4若有需要，请从同目录下加载，所需文件也请附上。
    （可不附）文章头图
    长宽可不等，文件名（不含扩展名）为 article，格式为 png、jpeg 或 svg。
    若未附上，则视为你同意由我来帮你的文章配图。
    （可不指定）CC 协议
    指定一种 CC 协议来共享你的文章。若不指定，视为你同意采用 CC BY-NC-SA 4.0 协议。
    • 向本站投稿，意味着你同意以这种 CC 协议向我无偿授权使用你的文章。当然，文章的版权仍归属于你。
    • 本站也有 CC BY-NC-SA 4.0 协议且由我原创的文章，可供你在 CC BY-NC-SA 4.0 协议下自由且无偿地使用，包括在你的网站上发表，所以我相信这是很公平的。

    资料之二：作者页面相关

    昵称/名字
    不实名或实名都可以。
    （可选）完整昵称/全名/超短描述
    它会出现在作者页面的昵称（名字）下方，以及作者列表页的昵称下方。
    正文
    Markdown 格式（不需要导言区），字数从 20 到 2000 字之间都可以。内容主要为你的个人介绍、网站介绍等。
    它会作为作者页面的正文部分。
    • 不可以在 Markdown 文件里嵌入需要从外部网站加载的资源，比如图片、CSS、JS 等。5
    • 其中必须包含你的个人网站名称及链接。6
    • （可选）如果你用分割线 --- 分隔出第一段（不可含有空行），我会让它以特殊字体出现在作者页面的开头，参见此页开头的那句话。
    • （可选）包含你的 GitHub/GitLab 或其它 Git 平台的主页地址，或者比如 Emacs China 之类的开源性质的社区的主页地址，以及邮箱等。
    你的头像
    作为你所投稿的文章的作者头像，也作为作者页面的作者头像，并且出现在作者列表页中。
    长宽相等，文件名（不含扩展名）为 avatar，格式为 png、jpeg 或 svg。
    请勿使用真人照片，除非可证明权利人是你自己，或你已获许可。
    （可选）一张稍大一些的图
    作为作者页面的配图（参见此页开头右侧的图）。
    长宽可不等，文件名（不含扩展名）为 featured，格式为 png、jpeg 或 svg。
    请勿使用真人照片，除非可证明权利人是你自己，或你已获许可。

    资料之三：意图以及所需资料

    必选项共 2 个。

    意图
    请说明你的具体意图，比如想要交换友链（当然啦），不然我会看着你邮件里的一堆资料而一头雾水。
    你需要我发送的资料
    我目前只接受双向友链，就是说需要你也在网站里加上我的友链。
    所以，还要请你在邮件里注明需要我发送的，作为友链相关的资料。
    作为交换，你也可以要求获得空间站内某篇采用 CC BY-NC-SA 4.0 协议共享且由我原创的文章的 Markdown 格式的文件。
    在投稿通过后，我会向你提供它们（涉及隐私等特殊情况除外）。

    交换友链之后的安排

    投稿通过后，在文章对应页面会出现其作者（你），并且在作者列表页也会出现你的作者页面卡片。7

    如果没有特殊情况的话，我会尽可能一直保留友链，一般不会主动取消。

    在取消友链时，除非我忘记了或者误操作，我一般会用邮件告知你。

    以下是对一些特殊情况的说明：

    • 如果我发现你的网站内容发生变化，并且感到我与你的网站格格不入（比如变成较大量地涉及饭圈、广告、民科等），可能会取消友链交换。当然这并不意味着对你的否定，我们只是在做双向选择而已。
    • 若我发现你的网站不再含有我的友链，我会发邮件询问相关情况。15 日内无答复则视为取消友链交换。
    • 若我发现你的网站无法访问，我会发邮件询问相关情况。超过 60 日无答复，我可能会取消友链交换。
    • 当友链取消时，作者页面会保留，但其上的网站链接可能不再保留。
    • 若你主动要求取消友链，你也可以要求我移除你的作者页面。
    • 你可以要求我移除你的投稿文章，我一般情况下会同意，但由于它以某个 CC 协议共享，我保留这个协议赋予我的继续发布它的权利。

    4. 单纯的外链并不需要从外部网站加载，所以不算这类资源。 ↩︎
